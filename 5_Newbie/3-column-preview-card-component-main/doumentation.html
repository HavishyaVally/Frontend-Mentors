3-Column Preview Card Component

Documentation & Technical Breakdown | Coded by Havishya Vally

Project Overview

This project is a solution to the 3-Column Preview Card Component challenge. The objective was to create a responsive card layout that adapts seamlessly from a vertical stack on mobile devices to a horizontal row on larger screens, while maintaining strict visual fidelity to the design mockups.

Layout Strategy

1. The Body Grid & Footer technique

Instead of using absolute positioning for the footer or complex flex calculations, I utilized a CSS Grid on the body. I defined a specific grid-template-rows structure: 1fr for the main content and 30px specifically reserved for the footer.

Combined with place-content: center and place-items: center, this ensures the card stays perfectly centered in the viewport, while the footer naturally sits at the bottom without overlapping. 

body {
    background-color: var(--page-bg);
    min-height: 100vh;

    /* Layout Engine */
    display: grid;

    /* 1fr for main content, 30px specific for footer */
    grid-template-rows: 1fr 30px;

    /* Perfect Center Alignment */
    place-content: center;
    place-items: center;
}

2. Responsive Container Logic

The .card-container uses a Mobile-First approach. It defaults to a single column stack. I identified 605px as the ideal breakpoint where the content has enough room to expand horizontally. At this point, the grid switches to 3 equal columns. 

.card-container {
    display: grid;
    grid-template-columns: 1fr;

    /* CSS Nesting for Breakpoint */
    @media (min-width: 605px) {
        grid-template-columns: repeat(3, 1fr);
    }
}

Component Logic

The "Flex-Margin" Alignment Trick

A common issue in card designs is that varying paragraph lengths cause the "Learn More" buttons to become misaligned.

My Solution: I applied display: flex and flex-direction: column to the individual card elements. By setting margin-top: auto on the button, it consumes all available vertical space above itself, forcing the button to the bottom of the container. 

button {
    margin-top: auto; /* Pushes button to bottom */
    padding: .5em 2em;
    width: fit-content;
    border: 2px solid white;
}

Typography Details

I customized the typography to ensure hierarchy. The headings use Big Shoulders Text (700), while the body text uses Lexend Deca (400) with a line-height of 1.8 for better readability compared to the default.
