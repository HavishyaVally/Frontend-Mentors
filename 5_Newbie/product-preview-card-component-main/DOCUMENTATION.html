Product Preview Card Component | Developed by Havishya Vally

My Approach

This project was built with a Mobile-First mindset. I focused on creating a clean, responsive card that adapts seamlessly from small screens to larger desktop views. The goal was to maintain high fidelity to the design while using modern, efficient CSS logic.

Design Philosophy: I utilized a semantic structure and a modular CSS approach. By defining a robust palette of variables, I ensured the project is easy to maintain and consistent with the brand colors provided in the style guide.

The Technical Implementation

1. Layout Strategy

To center the component perfectly on the screen, I used CSS Grid on the body. This is a modern alternative to older centering methods:

body {
    display: grid;
    place-content: center;
    min-height: 100vh;
    background-color: var(--page-bg);
}

2. Component Logic

The card is built using Flexbox. On mobile, the flex direction is set to column. For screens wider than 435px, it shifts to row. I utilized nth-child selectors for quick, efficient styling of the text elements within the card content without bloating the HTML with excessive classes. Although its not recommended, I did it for quick styling just for the challenge, not in real world implementation.

3. Modern Features

I implemented CSS Nesting to keep my media queries organized. Instead of having a separate block at the end of the file, the responsive logic lives right inside the class it modifies:

.card {
    display: flex;
    flex-direction: column;
    @media (min-width: 435px) {
        flex-direction: row;
    }
}

Key Challenges & Solutions

  • New Direction: I used the <picture> element to switch between the mobile and desktop images. This ensures the browser only loads the image size necessary for the user's device, optimizing performance.
  • Spacing: By using em units for gaps and margins, the internal spacing of the card feels fluid and proportional to the text size.
